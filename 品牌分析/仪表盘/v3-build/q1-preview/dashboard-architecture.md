# V3 仪表盘 · 架构设计契约（锁定后只准定点手术改）

> Terry 签字后锁定。Q1–Q6 全部照此模板；4 名常驻监工按此审（含结构一致性）。改任何一条须经 Terry。

## 一、全局外壳（一个页面装 6 个问题）

- **页宽**：内容区固定 **1440px 居中**；**所有 section 用同一容器宽度、同一左右边界**（杜绝宽度参差）。section 内部可再分栏，但外框一律对齐。
- **顶部固定区（全站只出现一次）**：品牌 + 报告标题 + 数据窗口（2025-02~2026-06）+ 全局来源图例（[SQP]=亚马逊搜索报告 / [领星]=后台实盘）。
- **吸顶导航**：`Q1 市场与季节｜Q2 份额｜Q3 角色｜Q4 漏斗｜Q5 抢词｜Q6 增长与护栏`。**Tab 切换**：一次只显示一个问题的完整模块（避免"满屏零散模块"）。当前 tab 高亮。

## 二、统一模块模板（每个 Q 都是这 6 层，顺序与视觉权重固定）

1. **问题头** —— Q 编号徽章 + 核心问题一句话 + 口径徽标（右对齐）。
2. **结论条** —— 一句白话结论，全模块最大字号（26px），全宽。
3. **关键指标** —— 3–4 个 KPI 卡，等宽一行，每个：指标名 / 大数 / 副标 + **来源标签**。
4. **图表区** —— 按§三图表规范；统一高度、统一配色、统一交互。
5. **解读与建议** —— 固定 3 张卡，等宽一行：`数据支撑`（蓝）/`专家建议`（橙）双芯片 + 标题 + 正文。
6. **审计层** —— `▸ 展开` 折叠；口径 / 算法 / 局限 / 不回答什么，可追溯到原始数据。

所有 Q 长同一个样 → 可预期、层级清晰、节奏统一。

## 三、图表规范（统一，按 Terry 意见）

- **领星销量**：**1 张图**，x 轴 **1–12 月**，两条线（2025 全年实线 / 2026 实线止 6 月）。同月两线看**同比**、各线走势看**环比**；`dataZoom` 默认停上半年、可拖看 11–12 月旺季峰。
- **SQP**：**需求侧（搜索）+ 成交侧（成交）各 1 张**，同款 12 月双线（可比老词口径为主）。
- **统一规则**：配色 我们=绿(#0f9d76) / 市场=slate(#5b6b8c) / 年份用实线(今年)+虚线(去年)；同一 tooltip、同一 dataZoom、同一坐标轴样式。**新词/长尾拆解用表格**（不占图）。

## 四、设计系统（治层级与节奏）

- **间距刻度**：8 / 16 / 24 / 32（内边距、块间距只用这几档）。
- **字号层级两档**：section 标题 18px（左侧色条）/ 卡内标题 15–16px；正文 15px；数字 `tabular-nums`。
- **颜色角色**：品牌蓝=交互/强调；绿=我们/领星；slate=市场/SQP；橙=专家建议；红=下降/警示。
- **标签统一**：解读区固定叫「解读要点」，折叠区固定叫「审计」——不再每处各写各的（如"两图解读要点""展开细节"）。
- **每个 section 外框对齐、视觉长度大致均衡**。
