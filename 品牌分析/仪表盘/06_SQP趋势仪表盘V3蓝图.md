# SQP 趋势仪表盘 V3 蓝图

本蓝图整合:5 个业务议题的正反辩论共识 + UX 部署版重设计方案 + 业务审查 + UX 审查 + 领星权威利润数据核实。所有关键数字均经子代理从原始数据(V2 内嵌 DATA、17 个月品牌视图/各 ASIN 视图 CSV、领星 MCP)独立复算。


## 〇、范围边界(已确认收口)

V3 的主干是 **SQP 搜索查询绩效情报**(关键词 × 漏斗 × 市场份额,即 Q1–Q6),这是这张表不可替代的价值。**利润是护栏(gate)、不是主干(spine)**:

- 利润护栏在 V3 里**薄而有界**——只做两样:① 每个运营组合的滚动**月度**毛利率 vs 7% 提成线(带退货率),用于给 SQP 的**付费冲量建议**亮红灯;② 单 SKU 促销深度天花板。
- **不**在这张表里做完整月度 P&L 拆解 / DSP 明细 / 退货归因——那是独立的财务看板,不混进 SQP 表。
- 已做的 10 个月运营提成 P&L 深挖,作为本蓝图第四节的**背景依据**保留,不占 V3 骨架。
- 7% 提成为**月度考核**(用户确认):月度口径下 1 月为结构性必失(12 月冲量的退货尾在 1 月),护栏的职责是**防复发**(别在 Q4 猛冲把某运营组合的当月提成埋雷),而非事后救火。近 3 月毛利回升(毛 13.7%/赵 15.3%)系**主动收走量强度+涨价+减促销**(用户确认),非淡季回血——即当前姿态毛利可持续。


## 一、五个必修的口径纠正(地基)

V3 不是"美化 V2",是先把五处口径病治好,再重建。

1. **解读与控件脱钩(结构 bug)**:`rerenderAll`(HTML 第403行)只刷新 KPI/图/表,不含 `renderInsight`;解读文案写死、永远停在单月 2026-06。切参考月/口径 → 数字与结论打架。
2. **动态关键词篮子致跨期不可比**:SQP 按词归集,每月 Top1000 词构成剧烈换血(相邻月 Jaccard 0.30–0.46)。跨月裸求和/相减(如"未捕获 324,349 单/96%")在数学上无效,必须落到可比篮子 + 披露覆盖率。
3. **中位价字段两套、易混**:DATA 里的聚合 `price_mine` 是**坏数据**(被 X1 污染到 $1,106–$3,348,对 $240–700 产品不可能);而各 ASIN 视图 col28/col29 是**干净**的。V3 只渲染干净的 per-ASIN 字段,禁渲染聚合坏字段。
4. **单 Hero 口径错误**:仪表盘把 B0DHPN1DMJ(S2)当"唯一 Hero/顶梁柱",事实错误——S2 非真爆款。须重建为 5 ASIN × 角色矩阵。
5. **零利润维度**:V2 全文毛利=0、净利=0、盈亏=0 次出现,却建议大力投放/Coupon/备货。必须补一层利润护栏(数据源见第三节)。

## 四、利润护栏(V3 新增核心层)——考核单位=运营组合,门槛=7% 提成线

**核心口径(已确认)**:公司按**运营负责人整体毛利率 ≥ 7%** 发提成(整体 <7% 不发提成);毛利率口径 = 领星 profit_report 毛利率(已含 PPC+DSP+联盟)。所以 P&L / 优先级单位是**运营组合**,不是单 ASIN、不是父体。走量款(毛香明的 S2、赵健宏的 X2)刻意压薄是策略,靠同运营名下其他 ASIN 把组合拉过 7%。

**运营组合逐月毛利率(领星 profit_report,准确窗口 2025-08~2026-05)——7% 是紧约束、不是舒适缓冲:**

| 月份 | 赵健宏(X+S1) | 毛香明(S2+S3) |
| --- | --- | --- |
| 2025-08 | 5.2% | −2.8% |
| 2025-09 | 2.7% | −1.6% |
| 2025-10 | 8.1% | −1.4% |
| 2025-11 | 8.9% | 2.9% |
| 2025-12 旺季峰 | 12.9% | 7.2% |
| 2026-01 退货潮 | −9.4% | −15.0% |
| 2026-02 | 9.0% | 9.5% |
| 2026-03 | 6.4% | 1.7% |
| 2026-04 | 13.5% | 12.6% |
| 2026-05 | 23.6% | 21.2% |

分段:全10月 赵健宏 **9.02%** / 毛香明 **3.95%**;H1-2026 赵 7.03% / 毛 **6.38%(<7%)**;近3月(3-5月) 赵 15.3% / 毛 13.7%(明显回升)。

**结论纠正(此前拿 2026-05 单月说"+14/+17pp、稳过线"是错的):**
1. **毛香明组合大部分月份低于 7%、全10月仅 3.95%**——走量款薄利已把它整组拖到提成线下,不是"完全 afford"。赵健宏(9.02%)相对健康但不宽裕。
2. **1 月双双巨亏(退货潮,退款率 27-37%)**:12 月冲的量在 1 月退回、吃掉利润→判断须按"冲量月+退货尾"的区间,不能单看 12 月漂亮或 1 月难看。**退货是一等一的利润驱动,SQP 看不到、必须从领星补进护栏。**
3. **近 3 月强劲回升**,若可持续则趋势向好,但须先判断是"走量强度主动收了"还是"淡季自然回血"。

7% 护栏当前**真在咬**(尤其毛香明),V3 必须把它当活约束、配退货维度显示,而非"放心冲"。

**护栏 = 三把尺,优先级尺改成运营组合:**

1. **运营组合毛利率 vs 7% 提成线(优先级/预算主尺)**:每个增长/投放/促销动作,看它把该**运营组合整体毛利率**推向还是推离 7%。**毛香明组合当前多数月份已在 7% 线下(全10月 3.95%),属需主动监控/收敛的状态,不能默认放心冲**;赵健宏组合(9.02%)相对有余但 1 月也翻负。护栏要展示逐月组合毛利率对 7% 的滚动轨迹 + 退货率,而非单月快照。
2. **单动作边际贡献 ROI(预算去哪)**:设 ROI 下限,替代按单 SKU 毛利率一刀切。
3. **单 SKU 毛利率 → 促销深度天花板(防单品失血)**:任何 Coupon/折扣深度 < 该 SKU 毛利率并预留边际——这把尺仍在 SKU 级,防止把某个走量款促到运营组合都拉不动。

- **免费杠杆**(CRO/加购弃单挽回/listing/主图)对全 SKU 放开;**付费摊薄杠杆**(深 Coupon/Prime/DSP)按①②③过闸。
- **删除"别给 S2 冲量、优先 X1"这类判断**——那是错框架。S2/X2 冲量是本职,判断看**运营组合是否守住 7%**。
- **数据源绑领星 `profit_report`**(dimension=asin 汇总到运营 / dimension=parent_asin;`nexgaios-lingxing` MCP,sid=7481,准确窗口 2025-08~2026-05)。
- **V3 新增运营视图**:按运营负责人分组,每个运营看自己组合的 SQP 表现 + 组合毛利率 vs 7% 提成线 + 缓冲——贴合真实问责结构。
- **"S2 单台净利 $67"是错标**(=挂牌价×毛利率,税前毛利非净利);促销红线用实测每单贡献更准。


## 五、UX 层:部署版重设计规范(经 UX 审查回炉)

方案主体可采纳,以下为审查后的必改点。

**到位可直接采纳**:叙事解耦到 H1 锚点 + 控件降级为图表下钻 + 每图口径徽标;Q5 守抢扩双芯片;Q2 改 2×2 小多图;双轴改**指数化基期=100** + markLine:100(消除"随手挪右轴让两线贴合"的操纵);字号≥12px;响应式(`max-width` + `ResizeObserver`→`chart.resize()`);吸顶导航 + scrollspy + 回到顶部 + `scroll-margin-top`;tooltip 边界翻转;WCAG AA 骨架(role/aria-sort/:focus-visible/prefers-reduced-motion)。

**必须回炉**:
- **【结构·基调骨架】恢复"展开详情"作为审计层(V2 做对、V3 切片丢了要捡回)**。每个模块 = **两层**:①**顶层**——简洁、白话、可扫读(结论+关键数字),给 30 秒抓重点;②**展开详情=审计层**——点开后,**每个数字/时间范围/口径都能追到"它怎么来的、为什么这么比"**,给要核实的老板/分析师、也给想查的外行。术语/数字/时间范围的完整解释一律沉到审计层,顶层只留白话+tooltip。**尤其 run-rate 这类易误解的对比,审计层必须写清推导+一句点破"参照时间段不同、不是数据打架"**(例:+25.1%=2026年2–6月3,443÷2025年2–6月2,753;689台/月=2026H1÷5;873台/月=2025年7–10月÷4;并列是提醒同比的漂亮部分来自去年上半年低基数)。这一条同时解决"外行看不懂/以为报告自相矛盾"与"审计追溯",让顶层敢简洁。复制到 Q2–Q6。
- **【最严重】叙事数字自洽**:UX 草案自己在 Q4/Q6 混入单月数字(见 Q4 改法)。V3 承诺锚死 H1,就必须把每个叙事数字都在 H1 基上重算硬编码。
- **配色对比度**:Okabe-Ito 的 sky `#56B4E9`(2.31:1)、orange `#E69F00`(2.25:1)在白底 **未达自设的 3:1**。浅色主题把这两色压深或加深色描边/halo、线宽提到 2.5–3px 配深色圆点,深浅两套都跑对比度校验。
- **曝光/点击仍是两种蓝**(色相差 0°):靠 Q2 小多图不同框兜底;确保任何图例/图中二者不并列,否则曝光换非蓝色相(yellow `#F0E442` 或 reddish-purple `#CC79A7`)。
