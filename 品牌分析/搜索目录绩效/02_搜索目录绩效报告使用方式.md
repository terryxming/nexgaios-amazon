# 这份报告能用来干什么

先给一句话的核心判断，再展开。

**搜索目录绩效是亚马逊品牌分析里唯一「按你自己的每个 ASIN、给出完整搜索漏斗 + 成交金额」的第一方数据。** SQP 按*搜索词*看市场和份额、但没有营收；这张表按*你的 ASIN*看**曝光→点击→加购→成交的完整漏斗，还带搜索来源销售额($)**。它把"我这个链接在搜索里到底表现如何、赚了多少"一次说清，是三张搜索报告里**最贴近自家经营结果**的一张。它的所有用途都从"按 ASIN + 有营收"这个独特性里长出来。

## 一、最核心用途：按 ASIN 诊断自家搜索漏斗健康度

对每个 ASIN，横向扫一遍漏斗，定位它在**哪一环掉队**（注意这里是逐级口径：CTR=点击/曝光、转化率=成交/点击）：

| 症状 | 判断 | 运营动作 |
|---|---|---|
| 曝光量低 | 搜索里没露脸 → 自然排名+广告不够 | 加广告、埋词、优化自然排名 |
| 曝光够、**CTR 低** | 露了没人点 → 主图/价格/评分/标题不吸引 | 优化主图、补 Review、调价 |
| CTR 够、**加购/转化率低** | 点进来没成交 → 详情页/价格/评价/库存劝退 | 优化 A+、变体、价格、清差评 |

（注：表里只给 CTR=点击/曝光、转化率=成交/点击，**转化率是从点击直接跳到成交、跳过了加购环**；想单独看"加购→成交"在哪失血，要自己用 加购数/点击、成交/加购 算，本表不直接给。）

和 SQP 的漏斗诊断互补：**SQP 告诉你"在某个词上"哪一环掉队，这张表告诉你"这个 ASIN 整体"哪一环掉队**——先用这张表锁定问题 ASIN，再用 SQP 下钻到具体词。

## 二、独一份：用「搜索来源销售额」评估搜索渠道价值

这是别的 BA 报告都没有的字段。它让你能回答几个原本要拼数据才能算的问题：

- **每个 ASIN 靠搜索赚了多少**：搜索来源销售额直接给出搜索归因 GMV，看哪些 ASIN 高度依赖搜索、哪些主要靠其它入口（浏览/Deals/站外）。
- **搜索渠道占比**：把搜索来源销售额和领星全渠道销售额对比，得到"这个 ASIN 有多少钱是搜索带来的"——高度依赖搜索的 ASIN，搜索排名一掉就伤营收，要重点守。⚠️ 但两者**口径/时区/退款处理不同源**（本表是含广告的毛额、只算搜索结果页；领星是结算净额），直接相除得到的占比是**近似值、有系统性偏差**，别当精确数。
- **营收视角选战场**：单看单量会高估低价 ASIN 的重要性；这张表让你能按**营收**而不是**单量**排优先级。⚠️ 要营收就**直接取"搜索来源销售额"原列**，别用"单量×售价中位数"反推（配件/多价款误差可达约 48%）；且实测约 **47% 的 ASIN 成交=0**（多为长尾/配件），"按营收/依赖度"只对少数走量款成立，别对全表算。

## 三、组合/角色矩阵：5 个主力谁是流量入口、谁是营收担当

这张表天然按 ASIN 排开，正好用来看组合分工（对应 V3 的 5-ASIN 角色矩阵）：

- 把每个 ASIN 的**曝光 / 点击 / 成交 / 搜索来源销售额**并排，一眼看出谁是"高曝光高流量入口"（如走量款 S2/X2）、谁是"低量高单价的营收/毛利担当"（如 X1）。
- 结合领星毛利，能把"搜索带来的营收"和"这营收挣不挣钱"接起来——但**注意这张表本身没有成本/利润**，利润要从领星 `profit_report` 补（见项目记忆的 7% 提成护栏）。

## 四、价格带与配送时效洞察

- **售价中位数**（各阶段）：⚠️ 它是**曝光时点值、随促销波动、且含异常脏值**（实测有 $15384 级离谱值，见 [01_搜索目录绩效表结构和表字段解读.md](01_搜索目录绩效表结构和表字段解读.md)），别当稳定挂牌价直接引用；价格诊断价值不如 SQP 的"大盘中位价 vs 你"对比。
- **配送时效**（当日达/1天/2天各列）：看该 ASIN 的曝光/点击/成交里有多少集中在快速配送档。若高度集中，说明顾客对时效敏感，是"要不要上 FBA/多仓/旺季提前布货"的直接证据。

## 五、趋势与季节性（你有 24 个月，这是最该做的）

单月是快照，你手上是**24 个月的连续序列**（2024-07~2026-06，比 SQP 更长更全），价值在跨月对比：

- **ASIN 级搜索营收的季节性**：卡拉OK机在圣诞/节假日的搜索成交高峰，按 ASIN 看谁最吃旺季，提前备货布广告。
- **漏斗健康度的月度走势**：某 ASIN 的 CTR/转化率逐月下滑 = 竞争加剧或 listing 老化的预警。
- **新 ASIN 爬坡**：新变体/新品进榜后的曝光→成交爬升轨迹，判断新品起没起来。

## 与其它两张报告的分工

- **搜索目录绩效（本表）= 自家结果视角**：按你的 ASIN 看**完整漏斗 + 搜索营收**。没有大盘/份额，也不按词归因。
- **SQP 搜索查询绩效 = 市场与份额视角**：按词看**市场多大、你的份额、哪一环掉队**。有大盘对比，但没有营收。
- **热门搜索词 = 竞品视角**：按词看**谁在赢**（排名 + 前 3 品牌/商品）。

三张表配合：用搜索目录绩效**发现哪个自家 ASIN 漏斗/营收有问题** → 用 SQP **下钻到是哪些词、你在市场里份额如何** → 用热门搜索词**看这些词是被谁抢走的**。

## 要诚实面对的局限

用之前得知道它**不能**干什么，否则会误判：

- **只是「搜索结果页来源」**：曝光/点击/成交/营收只算**搜索结果页(SERP)**入口——连详情页 widget、浏览、类目、Deals、站外都不算（比"搜索"更窄），营收是全渠道的子集、**不等于领星销售额**；且含自然+付费混合、拆不开。
- **口径与 SQP 不同**：CTR=点击/曝光、转化率=成交/点击（逐级），别和 SQP 的÷搜索量口径直接互比。
- **没有大盘/份额/竞品**：看不到市场总量、你的份额、对手是谁——那要 SQP 和热门搜索词。
- **不按搜索词归因**：只到 ASIN 级，不知道是哪个词带来的成交。
- **没有成本/利润**：有营收但无成本，ROI/毛利要结合领星利润数据。
- **月粒度、月末才出**：反应慢，是战略复盘工具，不适合日常调价/调 bid。
- **simple 版无评分**：评级/评分中位数两列为空，需要评分维度要导 detail 版。

---

**一句话总结意义**：这份报告的价值在于它把你品牌的**每个 ASIN 变成一条带营收的搜索漏斗**——你能看清每个链接在搜索里露脸多少、被点多少、成交多少、赚了多少钱、从哪一环开始掉队，从而把 listing 优化、广告、备货、组合分工这些动作，都**对准到具体的 ASIN 和具体的环节**上。它是三张搜索报告里最贴近"自家经营结果"的一张：SQP 看市场、热门搜索词看对手、搜索目录绩效看自己。
