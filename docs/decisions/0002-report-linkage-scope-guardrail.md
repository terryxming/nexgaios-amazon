# 0002 · 报告联动范围收口：利润是护栏、不是主干

- 状态：已采纳（2026-07-09）
- 相关：[../lessons-learned.md](../lessons-learned.md)、V3 蓝图 `品牌分析/仪表盘/06_SQP趋势仪表盘V3蓝图.md`


## 背景

V3 建设中，从"补一个利润护栏"（5 个业务问题里的 I4，本是一个有边界的小模块）一路深挖成了 10 个月的运营提成 P&L 分析（父子体、逐月毛利、退货潮）。这些发现真实且有价值，但已长成一条独立的"财务/提成监控"支线，SQP 表里根本没有利润数据。


## 决策

- **V3 主干 = SQP 关键词 × 漏斗 × 市场份额情报**（Q1–Q6），这是这张表不可替代的价值。
- **利润在 V3 里保持"薄而有界"**：只两样——① 每个运营组合的滚动月度毛利率 vs 7%（带退货率），给 SQP 的付费冲量建议亮红灯；② 单 SKU 促销深度天花板。**不**在这张表里做完整 P&L / DSP 明细 / 退货归因。
- 后来（三报告联动）新增的"词级作战表"同理：若验证有价值，作为 **Q4/Q5 的增强模块**并入 V3，**不另起炉灶做独立交付物**。


## 理由

- SQP 的独特价值是关键词/漏斗/份额情报；把它做成月度提成监控器是另一个工具、另一个受众（财务/HR）。
- "别让护栏盖成主楼"：附加层长成主干会稀释主线、拖慢交付。


## 后果

- 利润深挖沉淀为蓝图附录/备注，不占 V3 骨架。
- 三报告联动的作战表定位为 Q4/Q5 增强候选（是否固化待 Terry 拍板）。
