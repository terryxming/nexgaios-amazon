# 0003 · 7% 提成口径：以运营组合为考核单位

- 状态：已采纳（2026-07-09，Terry 确认）
- 相关：[../project-context.md](../project-context.md)（含逐月毛利率表）


## 背景

分析盈利/优先级时，先后试过以"单 ASIN"、以"父体"为单位，都对不上 Terry 的心智口径（他说走量款 S2/X2"略亏"，但领星单 ASIN 口径下 S2 是 +16.6%）。


## 决策

- **P&L / 优先级 / 问责单位 = 运营负责人的整体组合**（横跨多个父体），不是单 ASIN、不是父体。
  - 毛香明组 = S2 家族 + S3 家族 + 儿童新品。
  - 赵健宏组 = X 家族 + S1。
- **7% = 发提成门槛、月度考核**：运营组合整体毛利率 < 7% 当月不发提成。
- **口径 = 领星 `profit_report` 毛利率**（已含 PPC+DSP+联盟广告，接近净利），不是更严的内部净利。
- 走量款（S2/X2）刻意压薄/略亏是运营决策；判断"该不该给它冲量"看**运营组合整体是否守住 7%**，不看单 ASIN 自身毛利。


## 理由

- 7% 是公司真实的提成发放规则，运营按组合考核；这才是"该不该冲量"的真问责单位。
- 走量款压薄靠组合内高毛利款拉过线——这是设计使然，不是缺陷。


## 后果

- 护栏主尺 = 运营组合滚动月度毛利率 vs 7% 缓冲（促销红线仍在 SKU 级防单品失血）。
- 曾拿 2026-05 单月误判"两人稳过线、缓冲足"——**错**，已推翻；7% 是紧约束（毛香明多数月在线下、全 10 月仅 3.95%），必须按组合、逐月/区间看。护栏职责 = 防 Q4 猛冲复发。
