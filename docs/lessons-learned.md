## SQP 跨期口径

- **动态关键词篮子**：SQP 按词归集，每月 Top1000 词构成剧烈换血（相邻月 Jaccard 0.30–0.46）。**不能跨月裸求和/相减**（如"未捕获 324,349 单/96%"数学上无效）。跨期必须落可比篮子 + 披露覆盖率。→ 已固化：Q1 切片示范（稳定篮子 236 词覆盖 H1 成交 91.2%）+ 00 总纲。
- **参照系必须季节匹配**：run-rate 踩过坑——2026 淡季比 2025 旺季前 = 假显萎缩。同季对比才对。
- **中位价字段两套**：聚合 `price_mine` 是坏数据（被 X1 污染到 $1,106–$3,348）；各 ASIN 视图 **col28/col29 购买中位价才干净**，只用逐词干净字段。
- **搜索查询得分（第1列）不是热度**：是"1=对本品牌最相关"的相关性排名，与搜索量几乎无关（相关系数 −0.06）。文件原生按此列升序排，不是按搜索量。→ 已固化：SQP `01_` 文档已纠错。


## 词级作战表（三报告联动）方法教训

初版（单月 2026-06）被两个子代理 + 复算证伪，核心坑：

- **别用单月**（尤其淡季）：2026-06 是全年谷底，大盘总成交比 2025-12 小 **7.4×**，机会单被系统性缩 7×。必须多月/稳定篮子或旺季口径。
- **价带闸别用购买中位价**：它是被便宜段压弯的混合中位（karaoke machine 大盘 $60），会把"我们已在其中卖 $300"的溢价尾判成"价带低→放弃"。改用**大盘点击中位价 + 我方实际成交中位价**。
- **竞品只能按 ASIN 归因**：热门搜索词"前3品牌"列漂移（同一 ASIN 24 月被贴 38 个品牌名），品牌名仅参考。
- **三报告都没有竞品价格** → "$240+ 溢价子段占大词多少"算不出，只能估。
- **被证伪的错结论**：不是"没词可攻"，而是"用最淡月 + 便宜价尺子把 30% 的成交池误判死刑"。真正增长在"已站着却没使劲"的大词（免费 CRO 抬份额）+ X2 增长引擎 + SB/DSP 再营销 + Bundle 桥价差。
- → 已固化：**作战表·定版**（滚动12月2025-07~2026-06，修 4 项：品牌词过滤/CRO剔除已领先词/趋势薄样本护栏/价格右尺换单ASIN成交价）。产出 `品牌分析/仪表盘/词级作战表_定版.csv` + `_老板决策版.csv`。核心结论：零广告CRO主攻 4 词（karaoke machine 最大池）、"加投主攻"=0、守·领先词 14、儿童=产品缺口 20 词、头号对手 CYY(B0D1VJMSQG)。**待决**：是否固化进 V3 的 Q4/Q5。


## Q1 市场诊断（CVR"背离"）修正（经三报告审查纠正）

初版诊断被子代理证伪、复算确认，修正结论（已写进 07 切片解读+审计层）：

- **"搜索涨、成交跌、CVR 2.52→2.06" 主要是 2026 年 2-3 月一次性搜索灌水**（2月1.62%/3月1.84%异常低，4-6月回~2.4%），**不是结构性萎缩**。→ 教训：**CVR/背离类结论必须逐月看、别信 H1 平均**。
- **成因三分**：同词转化↓ + 词构成变 + **新词稀释**（约41万搜索、CVR仅0.46%、占跌幅~46%、83%集中在2-3月）。旧"儿童泛需求灌水"假设被证伪（儿童仅占增长8%、CVR高达4.8%）。疑与 Amazon Rufus/AI 搜索泛匹配有关，**待查不咬死**。
- **我们自己健康（搜索目录绩效交叉验证）**：5主力 H1 成交+32%、营收+27%、自家漏斗CVR 3.09→3.65%（升）。→ 教训：**判"我们健不健康"用搜索目录绩效最直接**。
- **唯一真内部警报=CTR 下滑**：曝光+36%、点击仅+12%，CTR 1.20→0.99%。"$300机器摆进$60货架、被看到没被点"，**免费可修**（主图/价格呈现/评分）。
- **karaoke with lyrics 是护城河机器词**（X2是该词赢家榜第2），别当垃圾内容流量弃掉；youtube 才是内容/低价靶子，两词别一锅端。

## v3-build 构建管线现状（2026-07-10 闭环）

- **单向构建，不手改产物**：`build_all.py` 串起 `build_v3_data` → SQP digests → `build_q1_data` → HTML 模板注入 → 合同校验。`q1_slice_data.json` 的基座与派生块都可从上游重建，`07_V3_Q1切片.html` 只由模板生成。
- **外部快照也是输入契约**：领星历史数据固化在 `data/lingxing-snapshot/`，元数据必须记录来源、行数、口径与刷新障碍；在线接口暂不可用时使用已验证快照，不能把数字重新抄回 Python/HTML。
- **真实浏览器验收需要 HTTP 上下文**：直接打开 `file://` 会遇到浏览器安全与 URL 策略限制。正确做法是用仓库内置的 Python 静态服务启动 `127.0.0.1`，再验渲染、控制台、交互与响应式；该流程已固化到 `.claude/launch.json` 和 handoff。
- **确定性是数据管线的回归线**：同一输入连续构建两次，关键 JSON/CSV/HTML 的哈希必须不变；同时用业务基准锁定领星 `13,758→17,146` 和 Q4 `35,298→43,990`，防止“脚本能跑但口径悄悄漂移”。

## SQP 头条口径：稳定篮子 vs 全榜（2026-07-10 复核纠正）

**踩了自己写进禁止清单的坑**：24 代理研究 / 08 报告 / 蓝图的 SQP 头条同比，用了**每月全榜（动态换血）求和**，而非稳定篮子——正是"禁止踩坑清单 B：SQP 跨月不能裸求和、只用 stable_basket"。Codex 的审查子代理先发现，我用仓库 `v3_data.json` 独立复算确认。

- **数字差很大**（Feb–Jun core）：大盘搜索 全榜 **+13.4%** vs 稳定篮子 **−0.85%**；我方 SQP 归因成交 **+25.1%** vs **+13.6%**；成交份额 **+1.18pp（3.41→4.59）** vs **+0.70pp（3.07→3.77）**。稳定篮子（171/240 词）覆盖 2026 H1 大盘成交 **90.3%**（覆盖够高、非小样本）。
- **结论方向不倒**：**领星真实 +24.6% 独立、锚不动**；"市场没自然增长/我们逆势抢份额"成立且**更硬**（稳定篮子直接给"持平略降"，不必再绕"2–3 月灌水"）。
- **撤"SQP +25% ≈ 领星 +24.6% 双源互证"**——是巧合；稳定篮子上 SQP 仅 +13.6%，差额来自新词/长尾/非搜索渠道（这本身是有用发现，不是矛盾）。
- **教训：覆盖率 ≠ 口径**。稳定篮子不能只用来算覆盖率、头条却用全榜求和；跨期头条必须落在稳定篮子上重算，全榜降为覆盖率旁注。→ 已固化：08 §一/二、06 §二.7/§四 Q1·Q2 头条口径已纠正；**07 页面与 `build_q1_data.py` 构建链同理待改（Codex 工程侧）**。
